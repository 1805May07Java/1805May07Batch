Well, here we are talking about Structured Query Language, from now on referred to as SQL, being pronounced as sequel. If you heard the same thing both times the first one was S Q L as separate letters.

So what do we use this SQL for? Creating, Modifying, and Searching data that we have stored in a database. SQL is the language we use to talk to a specific kind of database called a relational database.

So the above List and Table show what a data base table might look like. A table is how a relational database organizes information, each column of the table is a different property or piece of information about an object or person. A row or record in a data base is a collection of information from each column that forms a picture about what is stored in that table. In this case there is stored the First and Last names of a person and their job.

That takes care of three of the columns but what is that ID column about? That is the relational part of relational database, it is what is known as the Primmary Key. The Primary Key is a unique identifier that allows searches through the database to tell records apart. In this case it is an arbitrarily assigned ID value that tells them apart. It is not always an ID number there are cases where a particular kind of data is naturally unique or constrained as unique such as an email addresses.

So how does a special column that helps tell things apart lead into a relational database? In more complex sets of tables you can represent a relationship between records by having the primary key from one table be a column in another table. This ties the information from those two tables together logically. An example would be if we added addresses to the above table and those addresses were the primary key in a table concerning buildings or houses. Those addresses would be what is known as a foreign key in the person table above.

Now before we leave let us look at some actual SQL statements.
select * from TABLE_NAME;
insert into TABLE_NAME values (...)
update TABLE_NAME set values where condition
These are some of the basic statements from on of the sub languages of SQL. Yes SQL has sublanguages, the simple statement forms above are some of the most commonly used statements. The Select statement is what is used to retrieve information from the database. The statement above has a wildcard * but that could instead be specific columns in a database. The insert statement is used to add new records to the database, and the update method changes records. These methods along with the unlisted delete method form the Data MAnipulation Language or DML.
