SQL Term Definitions


  • Candidate key - column or set of columns which can uniquely identify a DB record.
  • Primary Key - candidate key which Is unique & not null- should not be changed, nor should it be sensitive information.
  • Natural key- naturally occurring NN&U data field. i.e. SSN, passport no.
  • Surrogate key - DB generated PK
  • Referential integrity- ensuring that relationships between entities remain consistent i.e. lack of deletion of columns referenced by F.K.'s & references to fields that do not exist, etc.
  • Domain integrity- ensuring that all fields are of the allowed type & size.
