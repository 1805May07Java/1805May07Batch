SQL Concepts

Sublanguages

SQL can be broken down into 5 sub-languages that have different responsibilities for a RDBMS.

  • Database Query Language (DQL) - Retrieve data, sometimes confused or included into DML
  • Database Definition Language (DDL) - For creating and changing the structure of a table
  • Database Manipulation Language (DML) - For manipulating records inside a table
  • Database Control Language (DCL) - Controls who has access to the data
  • Transcation Control Language (TCL) - Used to commit and restore data

Constraints

Constraints place restrictions or rules on a table's column.

Primary Key Foreign Key Not Null
Default Unique Check
  • Primary Key - Uniquely identifies a record
  • Foreign Key - A column from another table; usually a primary key
  • Not Null - Indicates that this column cannot be empty
  • Default - A default value for the column if it is left empty
  • Unique - The column's value must be unique compared to other values in the same column
  • Check - Only allows certain values within a condition

To make a foreign key

CONSTRAINT FK_Name FOREIGN KEY (Column_Name) REFERENCES Persons(Table_Where_Column_Is)

DB Normalization

Normalization is for increasing efficiency and reducing redundancy. It comes down to breaking down tables into smaller tables, and splitting or deleting columns if they are not atomic or not necessary, respectively

There is also the concept of de-normalization, which involves going in the other direction. This is primarily to increase readability because tables may be overnormalized which makes managing more of a hassle.
