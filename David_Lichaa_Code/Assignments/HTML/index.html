SQL Concepts

SQL can be broken down into 5 sublanguages

  • DML - Data Manipulation Language
  • DDL - Data Definition Language
  • DQL - Data Query Language
  • DCL - Data Control Language
  • TCL - Transaction Control Language

Here are some example commands of the different sublanguages

DML DDL DQL DCL TCL
INSERT CREATE SELECT GRANT COMMIT
SELECT ALTER REVOKE ROLLBACK
UPDATE DROP SET SAVEPOINT
DELETE TRUNCATE

In computer science, ACID (Atomicity, Consistency, Isolation, Durability) is a set of properties of database transactions intended to guarantee validity even in the event of errors, power failures, etc. In the context of databases, a sequence of database operations that satisfies the ACID properties, and thus can be perceived as a single logical operation on the data, is called a transaction. For example, a transfer of funds from one bank account to another, even involving multiple changes such as debiting one account and crediting another, is a single transaction.

Columns can have have certain constraints, which are rules that are enforced on tables or Columns

  • NOT NULL - null elements are not permitted in the column
  • UNIQUE - repeated elements are not permitted in the column
  • PRIMARY KEY - entries in this column are uniquely identified by having this constraint
  • FOREIGN KEY - links entries by creating a relationship in this column with another column in a different or same table
  • CHECK - custom constraint can be defined
